Product Requirements Document (PRD): Numeri
Edisi: Gemini Flash MVP Hackathon (v0.1)
Tujuan: Proof of Concept (PoC) dalam 48 Jam
Core Tech: Electron, Node.js, Google Gemini 2.5 Flash
1. Overview
Numeri (Hackathon Edition) adalah tool "Vibe-Coding" sederhana untuk mengubah spreadsheet kosong menjadi struktur data (JSON) yang rapi menggunakan AI.
Alih-alih memaksa pengguna mengikuti standar akuntansi, Numeri membiarkan pengguna mendefinisikan kolom mereka sendiri (Custom Schema) di spreadsheet kosong, lalu menggunakan kecepatan dan kecerdasan Google Gemini 2.5 Flash untuk melengkapi data, menambahkan struktur, atau memodifikasi skema tersebut melalui percakapan bahasa alami.
2. Goal Hackathon
Membuktikan bahwa Gemini 2.5 Flash dapat bertindak sebagai "Data Architect" instan yang mampu:
1. Memahami konteks tabel hanya dari header kolom yang diketik user (tanpa setup rumit).
2. Memanipulasi struktur JSON lokal dengan latensi rendah (< 2 detik).
3. Memberikan pengalaman "Blank Canvas" yang ramah bagi pengguna non-teknis.
3. Scope MVP (Fitur Wajib)
Hanya 3 fitur inti yang akan dibangun dalam 48 jam:
1. Blank Project & Custom Grid:
   * Aplikasi dibuka dengan grid kosong (seperti Excel baru).
   * Baris pertama (Row 1) dikunci sebagai Header Definition.
   * User bebas mengetik header (misal: "Menu", "Harga", "Stok").
   * Sistem membaca Header ini sebagai Key untuk JSON schema.
2. Single-Shot Gemini Agent:
   * Menggunakan 1 System Instruction tunggal: "Act as JSON modifier based on user prompt & current schema."
   * Menerima input: JSON saat ini + User Prompt.
   * Menghasilkan output: JSON baru (Full Replacement) menggunakan fitur JSON Mode Gemini.
3. Simple Diff & Save:
   * Menampilkan preview sederhana: JSON Lama (Kiri) vs JSON Baru (Kanan).
   * Tombol "Accept" untuk langsung menimpa file lokal (fs.writeFileSync).
4. Out-of-Scope (Fitur yang Dibuang)
Fitur berikut DILARANG dikerjakan di versi Hackathon ini:
* ❌ GAAP Validator & Rules: Tidak ada validasi debit/kredit atau saldo normal akuntansi.
* ❌ Multi-File Indexing: Hanya membaca 1 sheet aktif yang sedang dibuka.
* ❌ Complex Staging: Tidak ada atomic write atau patch tree. File langsung ditimpa.
* ❌ IBM WatsonX: Diganti sepenuhnya dengan Google Gemini API.
* ❌ Security Keychain: API Key disimpan di .env lokal sederhana.
* ❌ Standard Reports: Tidak ada template Neraca/Laba Rugi otomatis.
5. User Flow Ringkas
1. Start: User membuka aplikasi -> Klik "New Blank Project".
2. Define: Tampil Grid Kosong. User mengetik Header manual: [ Nama Teman | Utang | Tanggal ].
3. Prompt: User mengetik di chat: "Buatkan dummy data 5 teman yang berutang, dan tambahkan kolom Status Lunas."
4. Process:
   * Node.js mengubah Grid menjadi JSON.
   * Node.js mengirim JSON + Prompt ke Gemini Flash API.
   * Gemini mengembalikan JSON baru berisi data dummy + kolom baru Status.
5. Review: User melihat Diff (Preview perubahan).
6. Commit: User klik "Accept". Grid di UI terupdate otomatis.
6. Functional Requirements (FR)
ID
	Requirement
	Deskripsi Teknis
	FR-01
	Header Detection
	Grid UI harus menganggap Baris 1 sebagai Schema Keys. Baris 2 dst adalah Data Values.
	FR-02
	Chat Interface
	Input text sederhana untuk mengirim instruksi ke Gemini.
	FR-03
	Gemini Integration
	Backend memanggil gemini-2.5-flash dengan responseMimeType: "application/json" untuk menjamin output valid.
	FR-04
	Schema Respect
	Gemini harus diperintahkan (via System Prompt) untuk tidak mengubah nama kolom user (Custom Schema) kecuali diminta eksplisit.
	FR-05
	JSON Diff
	Tampilan read-only sederhana membandingkan JSON.stringify(old) vs JSON.stringify(new).
	FR-06
	Save Logic
	Fungsi simpan sederhana yang menimpa file .json di disk lokal saat user setuju.
	7. Non-Functional Requirements (NFR)
* Localhost Only: Aplikasi berjalan full di laptop demo.
* Latency Target: Respon Gemini Flash < 3 detik (ini keunggulan Flash).
* Reliability: Jika JSON output Gemini rusak (jarang terjadi di mode JSON), tampilkan error sederhana "AI Error, Try Again".

10. Demo Script / Success Criteria
Skenario: "Catatan Kas Warung Kopi (Non-Akuntan)"
1. 00:00 - 00:30 (Blank Canvas):
   * Buka Numeri. Tampil Grid Kosong.
   * User mengetik Header: Menu, Modal, Jual.
   * Poin: Menunjukkan fleksibilitas (tidak dipaksa pakai "Debit/Kredit").
2. 00:30 - 01:30 (AI Generation):
   * User Prompt: "Isi dengan 5 menu kopi, dan tambahkan kolom Untung (Jual - Modal)."
   * Action: Gemini Flash memproses dalam < 3 detik.
   * Result: Tampil JSON Diff berisi data kopi dan kolom Untung yang sudah terhitung benar.
3. 01:30 - 02:00 (The Adjustment):
   * User klik "Accept". Grid terisi.
   * User Prompt lagi: "Ubah kolom 'Menu' jadi 'Nama Produk'."
   * Gemini merevisi header JSON. User Accept.
   * Closing: "Numeri dengan Gemini Flash: Membangun struktur data secepat kecepatan berpikir."
